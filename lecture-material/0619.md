**HTML5**는 **웹 표준**을 최신화하고, 웹 애플리케이션을 더 효율적으로 만들기 위해 여러 새로운 기능과 태그를 도입한 버전입니다. **시맨틱 태그**는 HTML5의 핵심 기능 중 하나로, 문서의 의미와 구조를 명확히 나타내는 태그들을 제공합니다. 이러한 태그는 접근성, SEO(검색 엔진 최적화), 유지보수 측면에서 많은 이점을 제공합니다. 여기서는 **HTML5 문서**의 구조와 **시맨틱 태그** 사용법, 그리고 **IE8 이하 버전**에서의 대응 방법을 다루겠습니다.

### 1. **HTML5 문서**

HTML5 문서의 기본 구조는 다음과 같습니다. `<!DOCTYPE html>`을 사용하여 HTML5 문서임을 선언하고, HTML 요소와 메타 정보들을 포함합니다.

#### HTML5 문서 구조

```html
<!DOCTYPE html>
<html lang="ko">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>HTML5 Document</title>
</head>
<body>
    <!-- 문서 내용 -->
</body>
</html>
```

* **`<!DOCTYPE html>`**: HTML5 문서임을 선언하는 태그입니다. 이는 웹 브라우저가 HTML5를 올바르게 렌더링하도록 지시합니다.
* **`<html lang="ko">`**: `lang` 속성은 문서의 기본 언어를 정의합니다. 여기서는 한국어(`ko`)를 설정했습니다.
* **`<meta charset="UTF-8">`**: 문자 인코딩을 UTF-8로 설정하여 다양한 언어를 지원합니다.
* **`<meta name="viewport" content="width=device-width, initial-scale=1.0">`**: 반응형 웹 디자인을 지원하는 메타 태그입니다. 화면 크기에 맞게 페이지가 자동으로 조정됩니다.
* **`<title>`**: 브라우저 탭에 표시되는 페이지 제목을 설정합니다.

### 2. **문서 구조를 위한 HTML5 시맨틱 태그**

**HTML5 시맨틱 태그**는 문서의 구조를 정의하는 데 중요한 역할을 합니다. 각 시맨틱 태그는 웹 페이지의 의미 있는 부분을 나타내므로, 검색 엔진과 사용자에게 더 잘 이해될 수 있습니다. 대표적인 시맨틱 태그는 다음과 같습니다:

#### 2.1. **`<header>`**

* 페이지나 섹션의 **헤더**를 정의합니다. 보통 로고, 내비게이션, 제목 등이 포함됩니다.

  ```html
  <header>
      <h1>웹 페이지 제목</h1>
      <nav>
          <ul>
              <li><a href="#">Home</a></li>
              <li><a href="#">About</a></li>
          </ul>
      </nav>
  </header>
  ```

#### 2.2. **`<footer>`**

* 페이지나 섹션의 **푸터**를 정의합니다. 주로 저작권, 연락처, 사이트 맵 등이 포함됩니다.

  ```html
  <footer>
      <p>&copy; 2025 My Website</p>
  </footer>
  ```

#### 2.3. **`<nav>`**

* 내비게이션 링크를 정의하는 태그로, 사이트의 메뉴나 내비게이션 바 등을 포함합니다.

  ```html
  <nav>
      <ul>
          <li><a href="#">Home</a></li>
          <li><a href="#">Services</a></li>
          <li><a href="#">Contact</a></li>
      </ul>
  </nav>
  ```

#### 2.4. **`<main>`**

* 문서의 **주요 콘텐츠**를 감싸는 태그로, 페이지에서 중요한 콘텐츠를 나타냅니다. 한 페이지에 하나만 사용해야 합니다.

  ```html
  <main>
      <section>
          <h2>About Us</h2>
          <p>We are a company...</p>
      </section>
  </main>
  ```

#### 2.5. **`<section>`**

* 문서의 **독립적인 섹션**을 정의합니다. 보통 제목을 포함하며, 여러 섹션으로 나누어 페이지를 구성할 수 있습니다.

  ```html
  <section>
      <h2>Our Services</h2>
      <p>We provide web development...</p>
  </section>
  ```

#### 2.6. **`<article>`**

* **독립적이고 자립적인 콘텐츠**를 정의하는 태그로, 블로그 게시물, 뉴스 기사 등이 이에 해당합니다.

  ```html
  <article>
      <h2>Latest News</h2>
      <p>This is the latest news article...</p>
  </article>
  ```

#### 2.7. **`<aside>`**

* 문서의 **부수적인 콘텐츠**를 정의합니다. 사이드바, 관련 기사, 광고 등을 포함하는 데 사용됩니다.

  ```html
  <aside>
      <h2>Related Articles</h2>
      <ul>
          <li><a href="#">Article 1</a></li>
          <li><a href="#">Article 2</a></li>
      </ul>
  </aside>
  ```

#### 2.8. **`<figure>` 및 `<figcaption>`**

* **이미지**나 **다이어그램** 등 콘텐츠에 대한 설명을 추가할 때 사용됩니다.

  ```html
  <figure>
      <img src="image.jpg" alt="Description of image">
      <figcaption>Image Caption</figcaption>
  </figure>
  ```

### 3. **IE8 이하 버전에서는 어떻게 하나요?**

**HTML5 시맨틱 태그**는 최신 브라우저에서 잘 지원되지만, **Internet Explorer 8 이하 버전**에서는 시맨틱 태그를 제대로 지원하지 않습니다. 이 경우, 시맨틱 태그들이 기본적으로 **block-level 요소**로 동작하지 않기 때문에, 이를 해결하기 위해 **JavaScript**를 사용하여 HTML5 시맨틱 태그를 인식할 수 있도록 도와줘야 합니다.

#### 해결 방법: **HTML5 Shiv**

**HTML5 Shiv**는 **IE8 이하**에서 HTML5의 새로운 시맨틱 태그를 인식할 수 있도록 해주는 **JavaScript 라이브러리**입니다. 이 라이브러리는 HTML5 태그들이 제대로 스타일링되고 동작하도록 만들어줍니다.

1. **HTML5 Shiv 스크립트 추가**: 아래와 같이 `<head>`에 추가하여 HTML5 시맨틱 태그를 지원할 수 있습니다.

   ```html
   <!-- HTML5 Shiv for IE8 and below -->
   <script src="https://cdnjs.cloudflare.com/ajax/libs/html5shiv/3.7.3/html5shiv.min.js"></script>
   ```

2. **사용법**:

   * 위 스크립트를 추가하면 **IE8 이하 버전**에서도 HTML5 시맨틱 태그가 제대로 작동합니다. 이 방식은 **모든 최신 브라우저**에서 동작하며, 구형 브라우저에서는 `div`와 같은 기본 블록 레벨 요소로 처리됩니다.

### **요약**

1. **HTML5 문서**는 `<!DOCTYPE html>`로 시작하여, 기본적인 HTML5 요소(`header`, `footer`, `nav`, `main`, `section`, `article`, `aside`, `figure`)를 포함합니다.
2. **시맨틱 태그**는 웹 페이지의 구조를 명확히 하고, SEO와 접근성을 향상시키는 중요한 역할을 합니다. 각 시맨틱 태그는 콘텐츠의 의미를 나타냅니다.
3. **IE8 이하** 버전에서는 HTML5 시맨틱 태그를 지원하지 않으므로, **HTML5 Shiv** JavaScript 라이브러리를 사용하여 이를 해결할 수 있습니다.

HTML5와 시맨틱 태그는 웹 개발에서 중요한 부분을 차지하며, 최신 브라우저에서는 강력한 기능을 제공하고, 구형 브라우저에서는 호환성 문제를 해결할 수 있도록 지원됩니다.


### 시맨틱 태그를 사용하는 장점

1. **의미가 명확함**: 각 태그가 어떤 역할을 하는지 명확하게 나타내므로, 페이지 구조가 보다 이해하기 쉬워집니다.
2. **웹 표준 준수**: 시맨틱 HTML은 웹 표준을 따르므로, 웹 페이지가 다양한 브라우저에서 일관되게 작동할 확률이 높습니다.
3. **SEO 최적화**: 검색 엔진이 페이지의 의미를 더 잘 파악하여, 웹 페이지의 랭킹을 높이는 데 유리합니다.
4. **접근성 향상**: 스크린 리더와 같은 보조 기술이 시맨틱 태그를 사용하여 콘텐츠를 정확하게 읽고 전달할 수 있습니다.

### 시맨틱 HTML을 사용하는 좋은 예

```html
<!DOCTYPE html>
<html lang="ko">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My Website</title>
</head>
<body>
    <header>
        <h1>My Website</h1>
        <nav>
            <ul>
                <li><a href="#home">Home</a></li>
                <li><a href="#services">Services</a></li>
            </ul>
        </nav>
    </header>

    <main>
        <section>
            <h2>About Us</h2>
            <p>We are a web development company.</p>
        </section>
        <article>
            <h3>How We Work</h3>
            <p>Our process starts with planning and ends with deployment.</p>
        </article>
    </main>

    <footer>
        <p>&copy; 2025 My Website</p>
    </footer>
</body>
</html>
```

### 결론

**시맨틱 HTML**은 웹 페이지의 의미와 구조를 명확하게 정의할 수 있는 중요한 방법입니다. 이를 사용함으로써 웹 페이지가 보다 접근 가능하고, SEO 최적화가 잘 되며, 다른 개발자와 협업할 때 가독성 또한 향상됩니다. 시맨틱 태그를 활용하는 것은 웹 개발에서 필수적인 부분으로, 웹 페이지의 품질을 높이는 중요한 요소입니다.





### **HTML5와 멀티미디어**

HTML5는 **멀티미디어** 기능을 지원하는 새로운 요소들을 도입하여, 웹 페이지에서 비디오, 오디오, 애니메이션 등 다양한 미디어 콘텐츠를 쉽게 구현할 수 있도록 도와줍니다. HTML5는 특히 **`<audio>`** 와 **`<video>`** 태그를 통해 웹 페이지에서 멀티미디어 콘텐츠를 제어하고 표시할 수 있습니다. 이 외에도 동영상을 대문처럼 활용하는 방법도 포함됩니다.

---

### 1. **웹과 멀티미디어**

**웹과 멀티미디어**는 웹 페이지에 텍스트와 이미지만을 넘어서, **오디오**, **비디오**, **애니메이션** 등을 포함한 다양한 콘텐츠를 제공하는 데 중요한 역할을 합니다. HTML5는 이러한 멀티미디어 콘텐츠를 웹에서 직접 삽입하고 제어할 수 있는 기능을 제공합니다.

* 웹 페이지에서 **멀티미디어** 콘텐츠를 추가하면 사용자에게 더 몰입감 있는 경험을 제공할 수 있습니다.
* **HTML5**는 **`<audio>`** 와 **`<video>`** 태그를 통해 별도의 플러그인 없이도 오디오와 비디오 파일을 재생할 수 있도록 지원합니다.

### 2. **오디오 & 비디오 재생하기**

HTML5는 **`<audio>`** 와 **`<video>`** 태그를 통해 오디오 및 비디오 파일을 쉽게 웹 페이지에 삽입하고 제어할 수 있습니다. 이 태그들은 다양한 속성을 제공하여 사용자가 원하는 방식으로 오디오와 비디오를 제어할 수 있게 합니다.

#### 2.1. **`<audio>` 태그**

**`<audio>`** 태그는 오디오 파일을 웹 페이지에서 직접 재생할 수 있도록 합니다. HTML5에서는 오디오 파일을 제어할 수 있는 다양한 속성들을 제공합니다.

**구문**:

```html
<audio controls>
    <source src="audio.mp3" type="audio/mp3">
    Your browser does not support the audio element.
</audio>
```

* **`controls`**: 재생, 일시 정지, 볼륨 조정 등의 기본 제어 버튼을 추가합니다.
* **`<source>`**: 오디오 파일의 경로와 형식을 지정합니다.
* **`type`**: 오디오 파일의 MIME 타입을 지정합니다.

**예시**:

```html
<audio controls>
    <source src="audio/song.mp3" type="audio/mp3">
    Your browser does not support the audio element.
</audio>
```

#### 2.2. **`<video>` 태그**

**`<video>`** 태그는 비디오 파일을 웹 페이지에 삽입하고 제어할 수 있는 기능을 제공합니다. 오디오와 마찬가지로 다양한 속성을 통해 비디오 파일을 제어할 수 있습니다.

**구문**:

```html
<video controls width="600">
    <source src="video.mp4" type="video/mp4">
    Your browser does not support the video tag.
</video>
```

* **`controls`**: 비디오의 재생, 일시 정지, 음소거, 전체화면 등의 기본 제어 버튼을 추가합니다.
* **`width`**: 비디오의 너비를 설정합니다.
* **`<source>`**: 비디오 파일의 경로와 형식을 지정합니다.

**예시**:

```html
<video controls width="800">
    <source src="video/sample.mp4" type="video/mp4">
    Your browser does not support the video tag.
</video>
```

### 3. **동영상으로 홈페이지 대문 만들기**

HTML5에서는 **동영상을 홈페이지의 대문**(hero section)으로 활용할 수 있습니다. **`<video>`** 태그를 사용하여 배경으로 비디오를 설정하거나, 방문자가 페이지에 들어왔을 때 자동으로 재생되는 동영상을 설정할 수 있습니다.

#### 3.1. **비디오 배경으로 설정하기**

페이지의 **배경**으로 동영상을 설정하여 시각적인 효과를 추가할 수 있습니다. **`<video>`** 태그의 `autoplay`, `loop`, `muted` 속성을 사용하여 페이지 로드 시 자동으로 비디오가 재생되도록 할 수 있습니다.

**구문**:

```html
<video autoplay loop muted>
    <source src="background-video.mp4" type="video/mp4">
    Your browser does not support the video tag.
</video>
```

* **`autoplay`**: 페이지가 로드되자마자 비디오가 자동으로 재생됩니다.
* **`loop`**: 비디오가 끝나면 자동으로 처음부터 반복됩니다.
* **`muted`**: 비디오의 소리를 꺼둡니다.

#### 3.2. **CSS로 스타일링하여 대문 영역에 배경 비디오 설정**

동영상을 **배경 비디오**로 설정할 때는 CSS로 비디오 크기와 위치를 조정할 수 있습니다. 보통 화면 크기에 맞게 비디오가 꽉 차도록 스타일링합니다.

**예시 코드**:

```html
<!DOCTYPE html>
<html lang="ko">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>동영상 대문 만들기</title>
    <style>
        /* 비디오 배경 설정 */
        .hero {
            position: relative;
            width: 100%;
            height: 100vh;
            overflow: hidden;
        }

        .hero video {
            position: absolute;
            top: 0;
            left: 0;
            width: 100%;
            height: 100%;
            object-fit: cover; /* 비디오가 화면을 채우도록 */
        }

        .hero-content {
            position: absolute;
            top: 50%;
            left: 50%;
            transform: translate(-50%, -50%);
            color: white;
            text-align: center;
            font-size: 2rem;
        }
    </style>
</head>
<body>
    <div class="hero">
        <video autoplay loop muted>
            <source src="video/background.mp4" type="video/mp4">
            Your browser does not support the video tag.
        </video>
        <div class="hero-content">
            <h1>Welcome to Our Website</h1>
            <p>Your journey starts here.</p>
        </div>
    </div>
</body>
</html>
```

* **`object-fit: cover`**: 비디오가 화면의 크기에 맞게 자동으로 조정되며, 비디오가 화면을 꽉 채우도록 합니다.
* **`position: absolute`**: 콘텐츠가 비디오 위에 중앙에 위치하도록 설정합니다.

#### 3.3. **비디오로 홈페이지 대문 만들기 팁**

* **최적화된 비디오 사용**: 대용량 비디오는 페이지 로딩 속도를 느리게 만들 수 있기 때문에, 비디오 파일을 최적화하여 용량을 줄이고, 여러 포맷(`mp4`, `webm`, `ogg`)을 제공하는 것이 좋습니다.
* **반응형 웹 디자인**: 비디오가 화면 크기에 맞게 자동으로 크기가 조정되도록 설정합니다. (`object-fit: cover` 또는 `background-size: cover`)
* **배경 비디오와 콘텐츠의 대비**: 콘텐츠가 비디오 배경과 잘 보이도록 텍스트 색상이나 그림자 등을 추가하여 가독성을 높입니다.

### **요약**

1. **오디오 & 비디오 재생하기**:

   * HTML5에서 **`<audio>`** 와 **`<video>`** 태그를 사용하여 오디오와 비디오 파일을 직접 웹 페이지에 삽입하고 제어할 수 있습니다.
   * **`controls`**, **`autoplay`**, **`loop`**, **`muted`** 등 다양한 속성을 통해 오디오와 비디오의 재생 방식을 조정할 수 있습니다.

2. **동영상으로 홈페이지 대문 만들기**:

   * **배경 비디오**를 활용하여 시각적으로 매력적인 대문을 만들 수 있습니다.
   * **`object-fit: cover`** 와 같은 CSS 속성을 사용하여 비디오를 화면 크기에 맞게 조정하고, **`autoplay`**, **`loop`**, **`muted`** 속성을 사용하여 자동으로 재생되도록 설정할 수 있습니다.

HTML5와 멀티미디어 태그를 활용하면 웹 페이지에 더욱 풍성하고 몰입감 있는 콘텐츠를 제공할 수 있습니다.






### **다재다능한 CSS3 선택자**

CSS3 선택자는 웹 페이지의 스타일을 적용할 HTML 요소를 **선택** 하는 방법을 다양하게 제공하며, 더 정교하고 강력한 스타일링을 가능하게 합니다. CSS3에서 추가된 선택자들은 특정 요소나 속성, 상태에 대한 스타일을 더 쉽게 적용할 수 있게 해줍니다. 여기서는 **연결 선택자**, **속성 선택자**, **가상 클래스와 가상 요소** 에 대해 설명합니다.

![](https://velog.velcdn.com/images/ses2201/post/3e17baf9-6b18-40eb-94e8-7c1928b28b9f/image.png)

```html
<!DOCTYPE html>
<html>
  <head>
    <title>DOM 구조 예시</title>
  </head>
  <body>
    <section>
      <h1>
        <strong>제목 강조</strong>
      </h1>
      <img src="image.jpg" alt="예시 이미지" />
      <div>
        <p><strong>첫 번째 단락 강조</strong></p>
        <p><strong>두 번째 단락 강조</strong></p>
        <p><strong>세 번째 단락 강조</strong></p>
      </div>
    </section>
  </body>
</html>
```

### 1. **연결 선택자 (Combinator Selectors)**

연결 선택자는 **서로 다른 관계에 있는** HTML 요소들을 선택하는 방법입니다. 여러 요소 간의 관계를 기반으로 스타일을 적용할 수 있습니다.

#### 1.1. **자식 선택자 (`>`)**

* **자식 선택자**는 **부모 요소**의 **직접적인 자식** 만을 선택합니다.

**예시**:

```css
div > p {
    color: blue;
}
```

이 스타일은 **`div`** 요소의 **직접적인 자식**인 **`<p>`** 요소에만 적용됩니다. 중첩된 `<p>` 요소에는 적용되지 않습니다.

#### 1.2. **후손 선택자 (공백)**

* **후손 선택자**는 부모 요소와 그 자식 및 자손 요소에 모두 스타일을 적용합니다.

**예시**:

```css
div p {
    color: red;
}
```

이 스타일은 **`div`** 내의 모든 **`<p>`** 요소에 적용됩니다. **직접적인 자식**뿐만 아니라 **자손**에 있는 모든 `<p>` 요소에도 적용됩니다.

#### 1.3. **형제 선택자 (`~`)**

* **형제 선택자**는 **같은 부모를 가진** 요소 중에서 **지정한 요소 뒤에 나오는 형제**만 선택합니다.

**예시**:

```css
h2 ~ p {
    font-size: 18px;
}
```

이 스타일은 **`h2`** 뒤에 나오는 모든 **`<p>`** 요소에 적용됩니다.

#### 1.4. **인접 형제 선택자 (`+`)**

* **인접 형제 선택자**는 **특정 요소 바로 뒤에 있는 첫 번째 형제**만 선택합니다.

**예시**:

```css
h2 + p {
    color: green;
}
```

이 스타일은 **`h2`** 바로 뒤에 위치한 첫 번째 **`<p>`** 요소에만 적용됩니다.

### 2. **속성 선택자 (Attribute Selectors)**

속성 선택자는 HTML 요소의 **속성값**에 따라 스타일을 적용할 수 있게 해줍니다. CSS3에서는 속성 선택자를 더 다양하게 사용할 수 있게 확장했습니다.

#### 2.1. **기본 속성 선택자**

* 특정 속성의 존재 여부에 따라 스타일을 적용합니다.

**예시**:

```css
input[type="text"] {
    border: 1px solid black;
}
```

이 스타일은 **`type="text"`** 속성이 있는 모든 **`<input>`** 요소에 적용됩니다.

#### 2.2. **속성 값이 특정 값으로 시작하는 선택자 (`^=`)**

* 속성 값이 특정 값으로 시작하는 요소를 선택합니다.

**예시**:

```css
a[href^="https"] {
    color: green;
}
```

이 스타일은 **`href`** 속성이 **`https`** 로 시작하는 모든 링크에 적용됩니다.

#### 2.3. **속성 값이 특정 값으로 끝나는 선택자 (`$=`)**

* 속성 값이 특정 값으로 끝나는 요소를 선택합니다.

**예시** :

```css
img[src$=".jpg"] {
    border: 2px solid red;
}
```

이 스타일은 **`src`** 속성이 `.jpg`로 끝나는 모든 이미지에 적용됩니다.

#### 2.4. **속성 값에 특정 값이 포함된 선택자 (`*=`)**

* 속성 값에 특정 값이 포함된 요소를 선택합니다.

**예시**:

```css
a[href*="example"] {
    color: blue;
}
```

이 스타일은 **`href`** 속성에 **`example`** 이라는 단어가 포함된 모든 링크에 적용됩니다.

#### 2.5. **여러 값이 있는 속성 선택자**

* 여러 값 중 하나를 만족하는 요소를 선택합니다.

**예시**:

```css
input[type="text"], input[type="email"] {
    background-color: lightgray;
}
```

이 스타일은 **`type`** 이 `text` 또는 `email`인 모든 **`<input>`** 요소에 적용됩니다.

### 3. **가상 클래스와 가상 요소 (Pseudo-Classes & Pseudo-Elements)**

CSS의 **가상 클래스**와 **가상 요소**는 특정 상태나 특정 위치에 있는 요소에 스타일을 적용할 수 있는 강력한 도구입니다.

#### 3.1. **가상 클래스 (Pseudo-Classes)**

가상 클래스는 **특정 상태**나 **상호작용**에 따라 스타일을 적용합니다.

* **`:hover`**: 사용자가 요소에 마우스를 올렸을 때 스타일을 적용합니다.

  ```css
  a:hover {
      color: red;
  }
  ```

* **`:focus`**: 입력 필드나 링크가 포커스를 받을 때 스타일을 적용합니다.

  ```css
  input:focus {
      border-color: blue;
  }
  ```

* **`:active`**: 요소가 활성화된 상태일 때 스타일을 적용합니다. (주로 클릭된 상태)

  ```css
  button:active {
      background-color: darkgray;
  }
  ```

* **`:nth-child()`**: 부모 요소의 자식 요소 중 특정 순서에 있는 요소를 선택합니다.

  ```css
  li:nth-child(odd) {
      background-color: lightgray;
  }
  ```

* **`:first-child`**: 부모 요소의 첫 번째 자식 요소를 선택합니다.

  ```css
  p:first-child {
      font-weight: bold;
  }
  ```

* **`:last-child`**: 부모 요소의 마지막 자식 요소를 선택합니다.

  ```css
  p:last-child {
      font-style: italic;
  }
  ```

  물론입니다! 아래는 **연결 선택자(Combinators)** 를 설명하고, 해당 HTML 구조와 함께 사용할 수 있는 **예제 CSS 선택자**들을 정리한 것입니다.

## ✅ 연결 선택자 종류 & 예제

### 1. **자식 선택자 (`A > B`)**

`A`의 **직계 자식**인 `B`를 선택

```css
section > h1 {
  color: blue;
}
```

→ `section`의 **직계 자식**인 `h1`에만 적용

---

### 2. **후손 선택자 (`A B`)**

`A`의 **모든 후손** 중 `B`를 선택

```css
section strong {
  color: red;
}
```

→ `section` 내부에 있는 **모든 `strong` 요소**에 적용 (자식/자손 포함)

---

### 3. **형제 선택자 (일반 형제, `A ~ B`)**

`A` 이후에 오는 **같은 부모를 가진 형제** 중 `B`를 선택

```css
h1 ~ img {
  border: 2px solid green;
}
```

→ `h1` 이후에 나오는 `img`에 적용 (같은 부모인 경우만)

---

### 4. **인접 형제 선택자 (`A + B`)**

`A` 바로 다음에 오는 **형제 요소 `B`** 를 선택

```css
h1 + img {
  width: 200px;
}
```

→ `h1` **바로 뒤에 나오는 `img`** 에만 적용

---

### 5. **전체 구조와 함께 쓰기 좋은 예시**

```css
div > p > strong {
  font-weight: bold;
  color: orange;
}
```

→ `div`의 자식 `p`의 자식인 `strong` 요소에 적용 (자손 관계 명확히 지정)

## ✅ 구조 선택자 예제

### 1. `:first-child`

부모의 **첫 번째 자식 요소** 선택

```css
p:first-child {
  color: red;
}
```

→ `div` 안에서 **첫 번째 자식**이 `p`일 경우에만 적용
※ `div > p` 구조일 때 첫 번째 `p`에 적용됨

---

### 2. `:last-child`

부모의 **마지막 자식 요소** 선택

```css
p:last-child {
  color: blue;
}
```

→ `div` 안의 **마지막 `p` 요소**에 적용됨

---

### 3. `:nth-child(n)`

**n번째 자식 요소**를 선택 (`1`부터 시작)

```css
p:nth-child(2) {
  font-weight: bold;
}
```

→ `div` 안에서 **두 번째 자식 요소**가 `p`일 때 적용됨

```css
p:nth-child(odd) {
  background-color: #f0f0f0;
}
```

→ 홀수 번째 `p` 요소에 적용 (1, 3, 5...)

```css
p:nth-child(even) {
  background-color: #e0e0ff;
}
```

→ 짝수 번째 `p` 요소에 적용 (2, 4, 6...)

---

### 4. `:nth-of-type(n)`

해당 **요소 타입 중에서 n번째**를 선택

```css
p:nth-of-type(2) {
  text-decoration: underline;
}
```

→ `p` 태그 중 **두 번째**인 요소에만 적용됨
(중간에 다른 태그가 끼어 있어도 `p` 기준으로 순서 셈)

## 🔍 비교 예시

```html
<div>
  <p>첫 번째</p>       <!-- :first-child & :nth-child(1) & :nth-of-type(1) -->
  <span>중간</span>
  <p>두 번째 p</p>     <!-- :nth-child(3), but :nth-of-type(2) -->
</div>
```

```css
p:first-child      { color: red; }    /* 적용 안됨 - 첫 자식은 p가 아님 */
p:nth-child(3)     { color: green; }  /* 3번째 자식이 p니까 적용됨 */
p:nth-of-type(2)   { color: blue; }   /* 두 번째 p니까 적용됨 */
```

#### 3.2. **가상 요소 (Pseudo-Elements)**

가상 요소는 **실제 존재하지 않는** 요소에 스타일을 적용하는 방법입니다. 주로 텍스트나 콘텐츠의 특정 부분에 스타일을 적용할 때 사용됩니다.

* **`::before`**: 요소의 **앞부분**에 콘텐츠를 삽입합니다.

  ```css
  p::before {
      content: "★ ";
      color: gold;
  }
  ```

* **`::after`**: 요소의 **뒤부분**에 콘텐츠를 삽입합니다.

  ```css
  p::after {
      content: " ★";
      color: gold;
  }
  ```

* **`::first-letter`**: 요소의 **첫 번째 글자**에 스타일을 적용합니다.

  ```css
  p::first-letter {
      font-size: 2em;
      font-weight: bold;
  }
  ```

* **`::first-line`**: 요소의 **첫 번째 줄**에 스타일을 적용합니다.

  ```css
  p::first-line {
      color: red;
      font-weight: bold;
  }
  ```

### **요약**

1. **연결 선택자**는 여러 HTML 요소 간의 관계에 따라 스타일을 적용하는 선택자로, 자식, 후손, 형제 요소 간의 관계를 설정할 수 있습니다. (`>`, 공백, `~`, `+`)
2. **속성 선택자**는 특정 HTML 요소의 **속성값**에 따라 스타일을 적용합니다. 예를 들어, 속성 값이 특정 값으로 시작하거나 끝나는 요소를 선택할 수 있습니다. (`[attribute="value"]`, `[attribute^="value"]`, `[attribute$="value"]`, `[attribute*="value"]`)
3. **가상 클래스**는 요소의 특정 **상태**에 스타일을 적용하고, **가상 요소**는 요소의 특정 부분에 스타일을 적용합니다. (`:hover`, `:focus`, `::before`, `::after`, `::first-letter`, `::first-line`)

이 선택자들을 활용하면 HTML 요소를 보다 정교하고 세밀하게 스타일링할 수 있습니다.

## ✅ 속성 선택자란?

HTML 요소의 **속성(attribute)** 값을 기준으로 요소를 선택하는 CSS 선택자입니다.

---

## ✅ 예제 HTML 구조

```html
<input type="text" name="username" />
<input type="password" name="userpass" />
<input type="checkbox" checked />
<a href="https://example.com" target="_blank">Example</a>
<img src="profile.jpg" alt="사용자 사진" />
```

---

## ✅ 속성 선택자 예제

### 1. `[속성]`

특정 속성이 **존재하는 요소** 선택

```css
input[checked] {
  outline: 2px solid green;
}
```

→ `checked` 속성이 있는 `input`에 적용됨 (예: 체크박스)

---

### 2. `[속성="값"]`

속성의 **값이 정확히 일치**하는 요소 선택

```css
input[type="password"] {
  background-color: lightyellow;
}
```

→ `type`이 `"password"`인 input 요소에만 적용됨

---

### 3. `[속성~="값"]`

공백으로 구분된 속성값 중 **하나가 일치**하면 선택

```html
<div class="btn primary large"></div>
```

```css
div[class~="primary"] {
  border: 1px solid blue;
}
```

→ class 속성에 `"primary"`가 **단어로 포함**되어 있으면 적용

---

### 4. `[속성|="값"]`

속성값이 `"값"` 또는 `"값-*" (값으로 시작해서 하이픈 포함)`일 때 선택 (주로 lang 속성)

```html
<p lang="ko">안녕하세요</p>
<p lang="ko-kr">한국어</p>
```

```css
p[lang|="ko"] {
  color: red;
}
```

→ `lang="ko"` 또는 `lang="ko-kr"`에 적용

---

### 5. `[속성^="값"]`

속성값이 **값으로 시작**하는 요소 선택

```css
a[href^="https"] {
  color: green;
}
```

→ `https`로 시작하는 링크에 적용 (보통 외부 링크)

---

### 6. `[속성$="값"]`

속성값이 **값으로 끝나는** 요소 선택

```css
img[src$=".jpg"] {
  border-radius: 50%;
}
```

→ `.jpg`로 끝나는 이미지에 적용

---

### 7. `[속성*="값"]`

속성값에 **값이 포함**되어 있는 요소 선택 (중간에 있어도 가능)

```css
a[href*="example"] {
  font-weight: bold;
}
```

→ `example`이 포함된 `href`에 적용

---

## 📌 실전 활용 팁

```css
/* placeholder가 포함된 input에 스타일 적용 */
input[placeholder*="이름"] {
  border-color: crimson;
}

/* 새 창으로 열리는 링크에만 스타일 적용 */
a[target="_blank"]::after {
  content: " 🔗";
}
```

## ✅ 1. ARIA 속성 선택자 예제

ARIA(Accessible Rich Internet Applications)는 **접근성 향상**을 위해 사용하는 속성입니다. 일반 속성과 동일하게 CSS 선택자로 사용할 수 있습니다.

### 🎯 예제 HTML

```html
<button aria-pressed="true">눌림</button>
<button aria-pressed="false">안 눌림</button>
```

### 🎨 CSS 예제

```css
button[aria-pressed="true"] {
  background-color: #007bff;
  color: white;
}

button[aria-pressed="false"] {
  background-color: white;
  color: #007bff;
  border: 1px solid #007bff;
}
```

→ `aria-pressed="true"`인 버튼에 강조 스타일을 주고, 그렇지 않으면 반대 스타일

---

## ✅ 2. `data-*` 속성 선택자 예제

`data-*` 속성은 개발자가 HTML 요소에 **임의의 데이터**를 저장하고 사용할 수 있도록 만든 속성입니다. JavaScript에서도 자주 활용되고, CSS 선택자에서도 사용할 수 있어요.

### 🎯 예제 HTML

```html
<div data-role="admin">관리자</div>
<div data-role="user">사용자</div>
<div data-role="guest">게스트</div>
```

### 🎨 CSS 예제

```css
div[data-role="admin"] {
  background-color: #ffdddd;
  font-weight: bold;
}

div[data-role="user"] {
  background-color: #ddffdd;
}

div[data-role="guest"] {
  background-color: #ddddff;
  font-style: italic;
}
```

→ 각 역할에 따라 다른 스타일 적용

---

### 🎯 동적 UI 예제

```html
<button data-status="loading">로딩 중...</button>
<button data-status="done">완료</button>
```

```css
button[data-status="loading"] {
  opacity: 0.5;
  cursor: wait;
}

button[data-status="done"] {
  background-color: green;
  color: white;
}
```

→ `data-status` 속성 값을 통해 버튼 상태에 따른 스타일 지정

---

## 🔧 JavaScript 연계 예시

```js
const adminDiv = document.querySelector('[data-role="admin"]');
console.log(adminDiv.dataset.role); // "admin"
```

→ `dataset` 객체를 통해 `data-*` 값에 접근 가능

### **CSS3와 애니메이션**

**CSS3와 애니메이션**은 웹 페이지에 동적인 효과를 추가하는 데 중요한 역할을 합니다. **변형**(transform)과 **트랜지션**(transition), **애니메이션**(animation) 등을 사용하여 사용자 경험을 향상시키고, 시각적으로 매력적인 콘텐츠를 만들 수 있습니다. 아래에서는 **변형**, **변형과 관련된 속성들**, **트랜지션**, **애니메이션**에 대해 자세히 설명하고, **상품 페이지에 가격 떠오르게 만들기** 예제를 제공합니다.

[css 생성기](https://www.toptal.com/developers/css3maker)

---

### 1. **변형 (Transform)**

**`transform`** 속성은 요소의 모양이나 위치를 변경할 때 사용됩니다. 변형은 요소의 크기, 위치, 회전 등을 바꿀 수 있습니다. 이 속성은 **2D** 및 **3D** 변형을 지원합니다.

#### 1.1. **주요 변형 함수**

* **`rotate()`**: 요소를 회전시킵니다. 단위는 **도(degrees)** 입니다.

  ```css
  div {
      transform: rotate(45deg);
  }
  ```

* **`scale()`**: 요소의 크기를 확대하거나 축소합니다.

  ```css
  div {
      transform: scale(1.5); /* 1.5배 확대 */
  }
  ```

* **`translate()`**: 요소를 X축과 Y축 방향으로 이동시킵니다.

  ```css
  div {
      transform: translate(50px, 100px); /* 50px 오른쪽, 100px 아래로 이동 */
  }
  ```

* **`skew()`**: 요소를 기울입니다.

  ```css
  div {
      transform: skew(20deg, 10deg); /* X축 20도, Y축 10도 기울기 */
  }
  ```

* **`matrix()`**: 2D 변형을 한 번에 정의하는 함수로, 행렬을 사용하여 변형을 적용합니다.

#### 1.2. **예시**: 회전, 이동 및 크기 조정을 결합한 변형

```css
div {
    transform: rotate(45deg) translate(100px, 50px) scale(1.2);
}
```

이 예시에서는 **45도 회전**, **100px 오른쪽 이동**, **1.2배 확대**된 변형을 동시에 적용합니다.

---

### 2. **변형과 관련된 속성들**

변형을 사용할 때 관련 속성들을 함께 사용하여 애니메이션 효과를 적용하거나 스타일을 수정할 수 있습니다. 변형을 사용할 때 자주 사용되는 속성은 다음과 같습니다:

#### 2.1. **`transform-origin`**

* 변형이 적용되는 **기준점**을 설정합니다. 기본적으로 `center center`로 설정되어 있지만, 이 속성으로 기준점을 변경할 수 있습니다.

  ```css
  div {
      transform-origin: top left; /* 왼쪽 상단을 기준으로 변형 */
      transform: rotate(45deg);
  }
  ```

#### 2.2. **`perspective`**

* 3D 변형 효과를 적용할 때 원근감을 설정합니다. 이 속성은 부모 요소에 적용하여 자식 요소가 3D 효과로 변형될 때 깊이를 조정합니다.

  ```css
  .container {
      perspective: 500px;
  }
  .box {
      transform: rotateY(45deg);
  }
  ```

---

### 3. **트랜지션 (Transition)**

**트랜지션**은 요소의 스타일이 변경될 때 그 변화를 **부드럽게** 나타내는 효과입니다. 트랜지션은 **상태 변화**(예: 마우스 오버, 클릭 등)에 따라 요소의 스타일을 자연스럽게 변환할 수 있게 합니다.

#### 3.1. **`transition` 속성**

* `transition` 속성은 **변경될 속성**, **변화 시간**, **타이밍 함수**, **지연 시간**을 설정할 수 있습니다.

**구문**:

```css
element {
    transition: property duration timing-function delay;
}
```

* **`property`**: 변화를 적용할 속성입니다.
* **`duration`**: 변화가 일어날 시간입니다. 예: `1s` (1초), `500ms` (500밀리초).
* **`timing-function`**: 변화를 위한 타이밍 함수입니다. `ease`, `linear`, `ease-in`, `ease-out`, `cubic-bezier` 등이 있습니다.
* **`delay`**: 변화가 시작되기 전에 기다릴 시간입니다.

#### 3.2. **예시: 색상 변화 트랜지션**

```css
button {
    background-color: blue;
    transition: background-color 0.5s ease;
}

button:hover {
    background-color: red;
}
```

* 이 예시에서, 버튼의 배경색은 0.5초 동안 **부드럽게** 파란색에서 빨간색으로 변합니다.

#### 3.3. **트랜지션을 여러 속성에 적용하기**

```css
div {
    width: 100px;
    height: 100px;
    background-color: blue;
    transition: width 1s, height 1s, background-color 1s;
}

div:hover {
    width: 200px;
    height: 200px;
    background-color: red;
}
```

* **`div`** 요소는 마우스를 올리면 크기와 색상이 **부드럽게** 변경됩니다.

---

### 4. **애니메이션 (Animation)**

**CSS 애니메이션**은 요소가 **시간에 따라 변하는** 스타일을 정의할 수 있게 해줍니다. 애니메이션은 **`@keyframes`** 규칙을 사용하여 요소의 상태 변화를 지정하고, **`animation`** 속성으로 이를 적용합니다.

[애니메이션 완벽 가이드](https://www.heropy.dev/p/26YKpK)

#### 4.1. **`@keyframes`** 규칙

* **`@keyframes`** 는 애니메이션의 **스타트**와 **엔드** 상태를 정의하는 규칙입니다. 애니메이션의 중간 상태도 정의할 수 있습니다.

**구문**:

```css
@keyframes animation-name {
    from {
        /* 초기 상태 */
    }
    to {
        /* 끝 상태 */
    }
}
```

* 또는 **중간 상태**를 정의할 수 있습니다:

```css
  @keyframes example {
      0% { background-color: red; }
      50% { background-color: yellow; }
      100% { background-color: green; }
  }
```

#### 4.2. **`animation` 속성**

* **`animation`** 속성으로 애니메이션을 적용합니다. `animation` 속성에는 **이름**, **지속 시간**, **타이밍 함수**, **반복 여부** 등을 설정할 수 있습니다.

**구문**:

```css
element {
    animation: animation-name duration timing-function delay iteration-count direction;
}
```

#### 4.3. **애니메이션 예시**

```css
@keyframes move {
    from {
        left: 0;
    }
    to {
        left: 100px;
    }
}

div {
    position: absolute;
    width: 50px;
    height: 50px;
    background-color: red;
    animation: move 2s infinite alternate;
}
```

* 이 예시에서, **`div`** 요소는 **2초** 동안 **좌측에서 우측**으로 이동하며, **무한 반복**되고 **애니메이션 방향이 바뀌는** 효과를 가집니다.

---

### 5. **상품 페이지에 가격 떠오르게 만들기**

**목표**: **CSS 애니메이션**을 사용하여 상품 페이지에서 가격이 **떠오르는** 효과를 구현합니다.

#### 문제:

1. 상품 이미지와 가격을 포함하는 **`div`** 요소를 만들고, 기본적으로 가격은 **투명**하게 설정합니다.
2. 상품 이미지에 마우스를 올리면 가격이 **부드럽게 떠오르는** 애니메이션을 적용하세요.
3. 가격은 **1초** 동안 서서히 **불투명**하게 변하도록 설정합니다.

**예시 코드**:

```html
<!DOCTYPE html>
<html lang="ko">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>상품 페이지</title>
    <style>
        .product {
            position: relative;
            width: 300px;
            height: 300px;
            background-color: lightgray;
            text-align: center;
            overflow: hidden;
        }

        .product img {
            width: 100%;
            height: 100%;
        }

        .price {
            position: absolute;
            bottom: 20px;
            left: 50%;
            transform: translateX(-50%);
            font-size: 24px;
            color: white;
            opacity: 0;
            transition: opacity 1s ease;
        }

        .product:hover .price {
            opacity: 1;
        }
    </style>
</head>
<body>
    <div class="product">
        <img src="https://via.placeholder.com/300" alt="상품 이미지">
        <div class="price">$99.99</div>
    </div>
</body>
</html>
```

* **애니메이션 효과**: `opacity` 속성을 애니메이션화하여 가격이 서서히 나타나도록 설정했습니다. **`product:hover`** 상태에서 가격의 **불투명도**가 1로 변화하여 가격이 **떠오르는** 효과를 만들었습니다.

---

### **요약**

1. **변형 (Transform)**: CSS `transform` 속성을 사용하여 요소의 **회전**, **크기 조정**, **이동**, **기울기** 등을 적용할 수 있습니다.
2. **트랜지션 (Transition)**: 요소의 스타일 변화가 부드럽게 이루어지도록 하는 속성으로, 상태 변화 시의 변화를 **시간**을 두고 적용합니다.
3. **애니메이션 (Animation)**: `@keyframes`를 사용하여 요소의 스타일 변화를 **지속적**이고 **반복적으로** 적용할 수 있습니다.
4. **상품 페이지 가격 애니메이션**: CSS 애니메이션을 사용하여 상품 페이지에서 가격이 **떠오르는** 효과를 만들 수 있습니다.

이 기술들은 웹 페이지의 시각적 효과와 동적인 콘텐츠 구현에 매우 유용하게 활용됩니다.

# **반응형 웹이란?**

**반응형 웹 디자인**(Responsive Web Design)은 다양한 화면 크기와 해상도를 가진 장치에서 **웹 페이지의 레이아웃과 콘텐츠**가 자동으로 최적화되도록 만드는 웹 디자인 기법입니다. 반응형 웹 디자인을 사용하면, 웹 페이지가 **데스크탑, 태블릿, 모바일** 등 다양한 기기에서 동일한 품질로 표시되며, 사용자 경험을 향상시킬 수 있습니다.

반응형 웹을 구현하기 위한 핵심 기술은 **가변 그리드 레이아웃**, **미디어 쿼리**, **가변 레이아웃과 요소**입니다. 아래에서 이들에 대해 설명합니다.

![반응형 웹](https://www.poptin.com/blog/wp-content/uploads/2016/09/responsive.jpg)

---

### 1. **모바일 기기와 웹 디자인**

**모바일 웹 디자인**은 다양한 **스마트폰**과 **태블릿** 화면 크기에 맞춰 웹 페이지를 최적화하는 과정을 포함합니다. 기존에는 웹 디자인이 **데스크탑 중심**으로 이루어졌지만, **모바일 기기**의 사용 증가로 인해 **모바일 우선(mobile-first)** 디자인 접근 방식이 중요해졌습니다.

#### 모바일 기기에서 웹 디자인의 주요 요소:

* **디스플레이 크기**: 모바일 장치는 화면 크기가 작기 때문에, **모바일 웹 디자인**은 화면 공간을 효율적으로 활용해야 합니다.
* **터치스크린 인터페이스**: 모바일 기기에서 터치스크린을 사용하는 사용자 경험을 고려하여 버튼 크기, 링크 간격 등을 설계합니다.
* **빠른 로딩 속도**: 모바일 데이터 네트워크를 고려해 페이지 로딩 속도가 빠르고 효율적으로 최적화되어야 합니다.
* **자동 크기 조정**: **반응형 웹 디자인**을 적용하여, 화면 크기에 따라 **컨텐츠가 자동으로 조정**되도록 합니다.

---

물론입니다! 웹 개발에서 **viewport(뷰포트)** 는 **사용자가 보고 있는 화면 영역**을 의미하며, 특히 **반응형 웹 디자인(responsive design)** 에서 매우 중요한 개념입니다.

## ✅ 왜 중요할까?

모바일 기기에서는 화면 크기가 작기 때문에, 기본적으로 브라우저는 **웹사이트를 축소해서 보여줍니다.** 이걸 제어하려면 **viewport 메타 태그**를 설정해야 합니다.

## ✅ Viewport 설정 방법

HTML 문서의 `<head>`에 아래 메타 태그를 추가합니다:

```html
<meta name="viewport" content="width=device-width, initial-scale=1.0">
```

---

## ✅ 속성 설명

| 속성                   | 설명                      |
| -------------------- | ----------------------- |
| `width=device-width` | 기기의 화면 너리에 맞춰 콘텐츠를 표시   |
| `initial-scale=1.0`  | 초기 줌 레벨 설정 (1.0은 100%)  |
| `maximum-scale=1.0`  | 사용자가 줌을 못하게 제한할 때 사용    |
| `user-scalable=no`   | 핀치 줌(확대/축소)을 막고 싶을 때 사용 |

| 텍스트 작고 클릭 어려움         | 텍스트 크고 터치하기 쉬움  |

### 2. **가변 그리드 레이아웃 (Flexible Grid Layout)**

**가변 그리드 레이아웃**은 반응형 웹 디자인에서 가장 중요한 요소 중 하나로, **비율 기반**의 레이아웃을 사용하여 다양한 화면 크기에 맞게 콘텐츠가 **유동적으로** 변하도록 합니다.

#### 2.1. **그리드 시스템의 개념**

* **고정 크기** 대신 **비율(%)** 을 사용하여 각 요소의 크기를 설정합니다. 이렇게 하면, 요소가 부모 컨테이너의 크기에 비례하여 늘어나거나 줄어듭니다.

#### 2.2. **예시: 가변 그리드 레이아웃**

```css
.container {
    display: grid;
    grid-template-columns: 1fr 2fr 1fr;
    gap: 10px;
}

.item {
    background-color: lightblue;
    padding: 10px;
}
```

* 이 예시에서 **`grid-template-columns`** 속성은 3개의 열을 정의합니다. 첫 번째 열과 세 번째 열은 **1fr**, 두 번째 열은 **2fr**로 설정하여 두 번째 열이 두 배 더 넓어지도록 설정합니다.
* **`fr`**(fractional unit)은 화면 크기에 맞춰 가변적인 크기를 설정하는 유닛입니다.

#### 2.3. **가변 그리드와 미디어 쿼리**

* **미디어 쿼리**를 사용하여 화면 크기가 다를 때 **그리드 열의 개수**를 변경하거나, **아이템 크기**를 조정할 수 있습니다.

```css
@media screen and (max-width: 768px) {
    .container {
        grid-template-columns: 1fr;
    }
}
```

* 위 코드는 화면 크기가 **768px 이하**일 때, 그리드의 열을 **하나**로 설정하여 **세로 레이아웃**을 제공합니다.

---

### 3. **가변 레이아웃과 가변 요소 (Flexible Layout and Flexible Elements)**

반응형 웹 디자인에서는 **가변 레이아웃**과 **가변 요소**를 사용하여 콘텐츠가 **화면 크기에 맞게 유연하게** 조정되도록 합니다.

#### 3.1. **가변 레이아웃 (Flexible Layout)**

가변 레이아웃은 **픽셀 단위** 대신 **비율(%)**, **뷰포트 단위(vw, vh)**, **`flex`** 와 같은 **유연한 단위**를 사용하여 요소를 배치합니다.

* **`%`**: 요소의 크기를 부모 요소의 **비율**로 설정합니다.
* **`vw` (viewport width)**: 화면 너비의 1%에 해당하는 크기입니다.
* **`vh` (viewport height)**: 화면 높이의 1%에 해당하는 크기입니다.

#### 3.2. **`flexbox`를 이용한 가변 레이아웃**

**Flexbox**는 **가변적인 레이아웃**을 쉽게 구현할 수 있게 해주는 CSS 레이아웃 모델입니다. Flexbox를 사용하면 요소들이 **주어진 공간에 맞게 유동적으로** 배치됩니다.

```css
.container {
    display: flex;
    justify-content: space-between;
    flex-wrap: wrap;
}

.item {
    flex: 1 1 200px;  /* grow, shrink, basis */
    margin: 10px;
    background-color: lightcoral;
    padding: 10px;
}
```

* **`flex: 1 1 200px`** 는 요소가 **200px**로 시작하고, 부모 요소의 크기에 따라 유동적으로 크기가 변할 수 있도록 설정합니다.
* **`flex-wrap: wrap`** 은 자식 요소들이 화면 크기에 맞게 **줄바꿈** 되도록 합니다.

#### 3.3. **가변 요소 (Flexible Elements)**

요소를 **flexible**하게 만들기 위해, **`flex`**, **`min-width`**, **`max-width`** 등의 속성을 사용하여 **크기 조정**이 가능하도록 합니다.

* **`min-width`**: 요소가 **최소 크기**를 지정합니다.
* **`max-width`**: 요소가 **최대 크기**를 지정합니다.

```css
.item {
    min-width: 150px;
    max-width: 300px;
    width: 100%;
}
```

이 설정을 통해 **`<div>`** 요소는 최소 **150px**, 최대 **300px**의 크기로 **반응형**으로 크기가 조정됩니다.

---

### **결론**

**반응형 웹 디자인**은 다양한 **기기**에서 **최적화된 웹 경험**을 제공하기 위한 핵심 기술입니다. 이를 위해 다음과 같은 기법들이 사용됩니다:

1. **모바일 기기와 웹 디자인**: 모바일-first 접근 방식으로 **모바일 기기**에서도 잘 작동하도록 디자인합니다.
2. **가변 그리드 레이아웃**: **비율 기반**의 레이아웃을 사용하여 다양한 화면 크기에 적응할 수 있도록 설정합니다. `fr` 유닛을 사용한 그리드 레이아웃과 **미디어 쿼리**로 화면 크기에 맞춰 레이아웃을 변화시킬 수 있습니다.
3. **가변 레이아웃과 가변 요소**: **`flexbox`** 와 같은 기술을 사용하여 요소들이 **화면 크기**에 맞게 **유동적**으로 조정되도록 설정합니다.

이러한 기술들을 활용하면 **모바일, 태블릿, 데스크탑** 등 다양한 장치에서 일관된 사용자 경험을 제공할 수 있습니다.








### **미디어 쿼리 (Media Queries)**

### **CSS media 쿼리**

미디어 쿼리(Media Query)는 **화면 크기, 해상도, 장치 타입 등에 따라 다른 CSS 스타일을 적용**할 수 있도록 하는 CSS 기능입니다. 반응형 웹 디자인(responsive web design)을 구현할 때 핵심적으로 사용됩니다.

## ✅ 기본 문법

`0617/colorspace.html 을 복사해서 연습할 것`

```html
<meta name="viewport" content="width=device-width, initial-scale=1.0" />

      /* 데스크탑 */
      @media (min-width: 1025px) {
        .box {
          font-size: 20px;
          background-color: yellow;
        }
      }

      /* 태블릿 */
      @media (min-width: 768px) and (max-width: 1024px) {
        .box {
          font-size: 18px;
          background-color: blue;
        }
      }

      /* 모바일 */
      @media (max-width: 767px) {
        .box {
          font-size: 16px;
          background-color: red;
        }
      }

```

```css
@media media-type and (조건) {
  /* 조건이 참일 때 적용할 CSS */
}
```

예시:

```css
@media screen and (max-width: 768px) {
  body {
    background-color: lightblue;
  }
}
```

📌 위 코드는 화면 너비가 768px 이하인 **모바일/태블릿 화면**에서 배경색을 `lightblue`로 변경합니다.

### 1. **미디어 쿼리 이해하기**

**미디어 쿼리**는 **`@media`** 규칙을 사용하여 특정 조건을 만족하는 화면에서만 스타일을 적용할 수 있도록 하는 CSS 기능입니다. 예를 들어, 화면의 **너비**가 600px 이상일 때와 600px 이하일 때 서로 다른 스타일을 적용할 수 있습니다.

#### 1.1. **미디어 쿼리 구문**

```css
@media media-type and (condition) {
    /* CSS 규칙 */
}
```

* **`media-type`**: 미디어 유형을 정의합니다. 예: `screen`, `print`, `all` 등.
* **`condition`**: 조건을 설정합니다. 예: `min-width`, `max-width`, `orientation` 등.

#### 1.2. **조건 예시**

* **`min-width`**: 최소 너비 조건입니다. 이 조건은 화면이 **지정된 크기 이상**일 때 적용됩니다.
* **`max-width`**: 최대 너비 조건입니다. 이 조건은 화면이 **지정된 크기 이하**일 때 적용됩니다.
* **`orientation`**: 화면의 오리엔테이션이 **세로**(portrait)인지 **가로**(landscape)인지에 따라 스타일을 변경할 수 있습니다.

#### 1.3. **미디어 쿼리의 역할**

* **화면 크기**에 따라 다른 스타일을 적용합니다.
* **기기 해상도**, **오리엔테이션**에 맞춰 스타일을 적용합니다.

---

### 2. **미디어 쿼리 적용하기**

#### 2.1. **기본 예시**

```css
/* 기본 스타일 (모바일 우선) */
body {
    font-size: 16px;
}

@media screen and (min-width: 768px) {
    /* 화면 너비가 768px 이상일 때 */
    body {
        font-size: 18px;
    }
}

@media screen and (min-width: 1024px) {
    /* 화면 너비가 1024px 이상일 때 */
    body {
        font-size: 20px;
    }
}
```

* **기본 스타일**은 모바일 우선 디자인을 적용하고, **768px 이상**일 때 글꼴 크기를 **18px**로, **1024px 이상**일 때는 **20px**로 설정합니다.

#### 2.2. **기타 예시**

* **`min-width`와 `max-width`** 를 함께 사용하여 다양한 화면 크기에서 스타일을 다르게 적용할 수 있습니다.

```css
/* 기본 스타일 */
div {
    width: 100%;
}

@media screen and (max-width: 600px) {
    /* 600px 이하일 때 */
    div {
        background-color: lightblue;
    }
}

@media screen and (min-width: 601px) and (max-width: 1024px) {
    /* 601px ~ 1024px 사이일 때 */
    div {
        background-color: lightgreen;
    }
}

@media screen and (min-width: 1025px) {
    /* 1025px 이상일 때 */
    div {
        background-color: lightcoral;
    }
}
```

이 예시에서는 **3가지 화면 크기**에 맞춰 각각 다른 배경색을 적용합니다.

#### 2.3. **`orientation` 예시**

* **`orientation`** 속성을 사용하여 화면의 세로/가로 모드에 따라 스타일을 변경할 수 있습니다.

```css
@media screen and (orientation: portrait) {
    /* 세로 모드 */
    body {
        background-color: lightyellow;
    }
}

@media screen and (orientation: landscape) {
    /* 가로 모드 */
    body {
        background-color: lightgreen;
    }
}
```

* **세로 모드**에서는 **`lightyellow`** 배경색을 적용하고, **가로 모드**에서는 **`lightgreen`** 배경색을 적용합니다.

### 3. **미디어 쿼리를 사용해 웹 문서 만들기**

**미디어 쿼리**를 사용하여 다양한 화면 크기에서 적절한 레이아웃을 제공하는 **반응형 웹 문서**를 만들 수 있습니다. 이를 통해 페이지가 모바일, 태블릿, 데스크탑 등에서 자동으로 조정되도록 할 수 있습니다.

#### 3.1. **반응형 웹 페이지 만들기**

* 기본적인 **1열** 레이아웃을 사용하고, 화면 너비가 일정 크기 이상일 때 **2열** 레이아웃으로 변경되는 예시입니다.

**HTML**:

```html
<!DOCTYPE html>
<html lang="ko">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>반응형 웹 페이지</title>
    <link rel="stylesheet" href="styles.css">
</head>
<body>
    <header>
        <h1>반응형 웹 디자인</h1>
    </header>

    <div class="content">
        <div class="left-column">
            <h2>왼쪽 열</h2>
            <p>여기에는 왼쪽 콘텐츠가 들어갑니다.</p>
        </div>
        <div class="right-column">
            <h2>오른쪽 열</h2>
            <p>여기에는 오른쪽 콘텐츠가 들어갑니다.</p>
        </div>
    </div>

    <footer>
        <p>&copy; 2025 My Website</p>
    </footer>
</body>
</html>
```

**CSS**:

```css

        justify-content: space-between;
    }

    .left-column, .right-column {
        width: 48%; /* 두 열 레이아웃 */
    }
}

/* 미디어 쿼리 - 1024px 이상 */
@media screen and (min-width: 1024px) {
    .content {
        max-width: 1200px;
        margin: 0 auto;
    }
}
```

#### 3.2. **설명**

* **모바일 우선 디자인**: 기본적으로 **1열** 레이아웃을 사용하고, 화면 크기가 **768px 이상**일 때 **2열 레이아웃**으로 변경됩니다.
* **`max-width`** 속성을 사용하여 **1024px 이상**일 때 레이아웃의 **최대 너비**를 설정하고, 화면이 커지면 콘텐츠가 **중앙 정렬**됩니다.

---

### **요약**

1. **미디어 쿼리 이해하기**:

   * **미디어 쿼리**는 특정 조건(화면 크기, 해상도, 오리엔테이션 등)을 기반으로 스타일을 적용할 수 있게 해주는 CSS 기능입니다.
   * **`@media`** 규칙을 사용하여 조건에 맞는 스타일을 정의합니다.

2. **미디어 쿼리 적용하기**:

   * **`min-width`**, **`max-width`**, **`orientation`** 등을 사용하여 다양한 화면 크기에서 다른 스타일을 적용합니다.
   * **미디어 쿼리**를 사용하여 **반응형 웹 디자인**을 구현하고, 화면 크기에 따라 레이아웃을 조정할 수 있습니다.

3. **미디어 쿼리를 사용해 웹 문서 만들기**:

   * **반응형 웹 페이지**를 만들 때, **`@media`** 규칙을 사용하여 모바일, 태블릿, 데스크탑에서 각각 적합한 스타일을 적용할 수 있습니다.
   * **그리드 레이아웃**과 **플렉스박스**를 활용하여 유연한 레이아웃을 설계할 수 있습니다.

**미디어 쿼리**는 다양한 화면 크기와 장치에서 최적화된 사용자 경험을 제공하는 데 필수적인 도구입니다.





### **플렉스 박스 레이아웃 (Flexbox Layout)**

**플렉스 박스**(Flexible Box Layout)는 복잡한 레이아웃을 쉽게 만들 수 있는 CSS3의 레이아웃 모델로, 주로 **1차원 레이아웃**(가로 또는 세로)에서 요소들을 유동적으로 배치하고 정렬하는 데 사용됩니다. 플렉스 박스를 사용하면 **항목들의 크기**, **간격**, **정렬**을 쉽게 조정할 수 있습니다. 기존의 **`float`** 이나 **`position`** 을 사용한 레이아웃보다 훨씬 더 유연하고 효율적입니다.

[![Flexbox](https://miro.medium.com/v2/resize:fit:720/format:webp/0*YeaUsQyhXSL1TCTH.png)](https://medium.com/@MakeComputerScienceGreatAgain/understanding-flexbox-a-comprehensive-guide-992bcd5f04de)

---

### 1. **플렉스 박스 레이아웃과 기본 속성들**

**플렉스 박스**는 **부모 요소**(컨테이너)와 **자식 요소**(항목)로 나뉘며, 부모 요소에 `display: flex`를 설정하고, 자식 요소에 다양한 정렬 및 크기 조정을 설정할 수 있습니다.

#### 1.1. **`display: flex`**

* **부모 요소**에 `display: flex`를 설정하면, 자식 요소들이 **플렉스 항목**이 되어 **플렉스 컨테이너**로 작동합니다.

**예시**:

```css
.container {
    display: flex;
}
```

* 이렇게 설정하면, **`.container`** 요소의 자식들이 **플렉스 항목**이 되어 **가로 방향**으로 나열됩니다.

#### 1.2. **주축과 교차축 (Main Axis and Cross Axis)**

* **주축(Main Axis)**: 플렉스 항목들이 기본적으로 배치되는 방향입니다. `flex-direction`에 따라 가로(`row`) 또는 세로(`column`)로 배치됩니다.
* **교차축(Cross Axis)**: 주축과 수직인 축으로, 항목들이 교차축 방향으로 배치됩니다.

#### 1.3. **`flex-direction`**

* **`flex-direction`** 속성은 플렉스 항목들의 배치 방향을 정의합니다.

| 값                | 설명                         |
| ---------------- | -------------------------- |
| `row` (기본값)      | 항목들을 **가로 방향**으로 배치        |
| `column`         | 항목들을 **세로 방향**으로 배치        |
| `row-reverse`    | 항목들을 **가로 방향**으로 반대 순서로 배치 |
| `column-reverse` | 항목들을 **세로 방향**으로 반대 순서로 배치 |

**예시**:

```css
.container {
    display: flex;
    flex-direction: column;
}
```

* 이 예시에서는 **`.container`** 의 자식들이 **세로 방향**으로 배치됩니다.

#### 1.4. **`flex-wrap`**

* **`flex-wrap`** 은 플렉스 항목들이 **한 줄에 모두 배치될 수 없을 때**, 항목들을 **다음 줄로 넘겨줄지 여부**를 결정합니다.

| 값              | 설명                       |
| -------------- | ------------------------ |
| `nowrap` (기본값) | 모든 항목들이 **한 줄**에 배치      |
| `wrap`         | 항목들이 **여러 줄로 배치**        |
| `wrap-reverse` | 항목들이 **반대 순서**로 여러 줄로 배치 |

**예시**:

```css
.container {
    display: flex;
    flex-wrap: wrap;
}
```

* 이 예시에서는 항목들이 **여러 줄로 배치**됩니다.

#### 1.5. **`justify-content`**

* **`justify-content`** 는 **주축**(가로 또는 세로) 방향으로 플렉스 항목들을 **정렬**하는 속성입니다.

| 값               | 설명                                           |
| --------------- | -------------------------------------------- |
| `flex-start`    | 항목들을 **시작 위치**에 정렬 (기본값)                     |
| `flex-end`      | 항목들을 **끝 위치**에 정렬                            |
| `center`        | 항목들을 **중앙**에 정렬                              |
| `space-between` | 항목들 간에 **공간을 균등하게 분배** (첫 항목과 마지막 항목은 끝에 배치) |
| `space-around`  | 항목들 간에 **공간을 균등하게 분배** (항목 양옆에도 공간이 있음)      |
| `space-evenly`  | 항목들 간에 **균등한 공간**을 배분                        |

**예시**:

```css
.container {
    display: flex;
    justify-content: center;
}
```

* 이 예시에서는 플렉스 항목들이 **수평으로 중앙에 정렬**됩니다.

---

### 2. **플렉스 박스 항목 배치를 위한 속성들**

#### 2.1. **`align-items`**

* **`align-items`** 속성은 **교차축**(세로 또는 가로) 방향으로 플렉스 항목들을 **정렬**하는 데 사용됩니다.

| 값               | 설명                       |
| --------------- | ------------------------ |
| `stretch` (기본값) | 항목들을 **교차축 방향으로 늘려서** 정렬 |
| `flex-start`    | 항목들을 **교차축 시작 위치**에 정렬   |
| `flex-end`      | 항목들을 **교차축 끝 위치**에 정렬    |
| `center`        | 항목들을 **교차축 중앙**에 정렬      |
| `baseline`      | 항목들의 **기준선**을 맞춰서 정렬     |

**예시**:

```css
.container {
    display: flex;
    align-items: center;
}
```

* 이 예시에서는 플렉스 항목들이 **세로로 중앙에 정렬**됩니다.

#### 2.2. **`align-self`**

* **`align-self`** 는 개별 플렉스 항목에 대해 **교차축 방향**으로 정렬을 다르게 설정할 수 있게 해줍니다.

| 값            | 설명                         |
| ------------ | -------------------------- |
| `auto` (기본값) | 부모 요소의 `align-items` 값을 따름 |
| `flex-start` | 항목을 **교차축 시작 위치**에 정렬      |
| `flex-end`   | 항목을 **교차축 끝 위치**에 정렬       |
| `center`     | 항목을 **교차축 중앙**에 정렬         |
| `baseline`   | 항목을 **기준선**에 맞춰 정렬         |
| `stretch`    | 항목을 **교차축 방향으로 늘려서** 정렬    |

**예시**:

```css
.item {
    align-self: flex-end;
}
```

* 이 예시에서는 **개별 항목**이 **교차축 끝 위치**에 정렬됩니다.

#### 2.3. **`flex-grow`, `flex-shrink`, `flex-basis`**

* **`flex-grow`**: 항목이 사용할 수 있는 공간을 얼마나 확장할지 설정합니다.
* **`flex-shrink`**: 항목이 공간이 부족할 때 얼마나 줄어들지 설정합니다.
* **`flex-basis`**: 항목의 기본 크기를 설정합니다.

**예시**:

```css
.item {
    flex-grow: 1;
    flex-shrink: 1;
    flex-basis: 100px;
}
```

* 이 예시에서는 **`flex-grow`** 가 **1**로 설정되어 있어, 요소가 **남은 공간을 차지**하도록 확장됩니다.

---

### 3. **예시: 플렉스 박스를 활용한 레이아웃**

```html
<!DOCTYPE html>
<html lang="ko">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>플렉스 박스 레이아웃</title>
    <style>
        .container {
            display: flex;
            justify-content: space-between;
            align-items: center;
            height: 200px;
            background-color: #f4f4f4;
        }

        .item {
            width: 30%;
            height: 100px;
            background-color: #4CAF50;
            color: white;
            text-align: center;
            line-height: 100px;
        }
    </style>
</head>
<body>
    <div class="container">
        <div class="item">항목 1</div>
        <div class="item">항목 2</div>
        <div class="item">항목 3</div>
    </div>
</body>
</html>
```

이 예시에서는 **플렉스 컨테이너**를 사용하여 **3개의 항목**을 **수평으로 정렬**하고, 항목들 사이에 **균등한 공간**을 배분했습니다.

---

### **요약**

1. **플렉스 박스 레이아웃**:

   * **`display: flex`** 를 사용하여 부모 요소를 플렉스 컨테이너로 설정하고, 자식 요소는 플렉스 항목이 됩니다.
   * **`flex-direction`** 을 사용하여 요소들의 배치 방향을 설정합니다. (가로 또는 세로)

2. **플렉스 박스 항목 배치를 위한 속성들**:

   * **`justify-content`**: 항목들을 주축 방향으로 정렬합니다.
   * **`align-items`**: 항목들을 교차축 방향으로 정렬합니다.
   * **`flex-grow`**, **`flex-shrink`**, **`flex-basis`** 등을 사용하여 플렉스 항목의 크기와 확장/축소 동작을 설정합니다.

플렉스 박스를 활용하면 다양한 **레이아웃**을 손쉽게 구현할 수 있으며, **반응형 디자인**에도 매우 유용합니다.
